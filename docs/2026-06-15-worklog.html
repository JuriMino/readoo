発生したエラー / つまずき

エラー① npm run build の後に npm run dev したら表示が崩れた

原因： npm run dev で public/hot が生成され、 @vite が「Vite開発サーバー（localhost:5173）から資産を取る」モードに切替。 しかしVite開発サーバーはappコンテナ内で動いており、ポート未公開のためホスト側ブラウザから届かずCSS/JS取得に失敗。

解決： vite.config.js に host: '0.0.0.0' を追加し、 compose.yaml で 5173:5173 を公開、コンテナ再作成。

エラー② ポート追加後も localhost が開けず、8080は崩れたまま

原因： compose.yaml への ports 追記がまだで、 かつコンテナが古いまま（再作成していなかった）。docker compose ps で app に 5173 が公開されていないことを確認。public/hot は http://localhost:5173 を指すが接続不可（HTTP 000）。

解決： compose.yaml に ports: "5173:5173" を追記 → docker compose up -d で再作成 → npm run dev 再実行で解消。

主な質問と回答（メモ）

Q. Breezeを入れれば主要なログイン機能は実装できる？

A. おおむねYes。登録・ログイン・ログアウト・プロフィール編集・退会の画面とロジックが生成される。 ただしBreezeは「足場」なので、論理削除対応・パスワードリセットのスコープ調整・ブランドカラー適用などは別途必要。

Q. 5173 はBreezeの指定ポート？

A. いいえ。Vite（フロント資産のビルド/開発サーバー）のデフォルトポート。 vite.config.js に server 記述が無ければデフォルトの 5173 / localhost で起動する。

Q. softDeletes() をマイグレーションに入れれば論理削除になる？

A. それだけでは不十分。deleted_at カラムが作られるだけで、実際に論理削除として動かすには User モデルへの SoftDeletes トレイト適用が必要（本日対応済み）。
