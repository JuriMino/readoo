作業手順（段階的に進める）

進め方の原則：一気に作らず、各ステップごとにブラウザで表示確認する。崩れたらどのステップが原因か切り分けやすい。

0

準備：ブランチ作成・色定義

  • 作業ブランチを切る（例：feature/(issue番号)-auth-ui）
  • tailwind.config.js に base: '#FAF6EC' を追加（クリーム背景用）
  • npm run dev を起動しておく（保存で自動反映）
1

共通レイアウトの枠を組む（guest.blade.php）

中央寄せカードだけの構造を、ヘッダー / メイン / フッターの縦3分割に組み替える。

<body class="min-h-screen flex flex-col bg-base">
    <header> 青いナビ（ロゴ ＋ Register｜Log in リンク）</header>

    <main class="flex-1 ...">
        {{ $slot }}   <!-- login / register がここに入る -->
    </main>

    <footer> 青いバー（© 2026 Readoo!…）</footer>
</body>
  • min-h-screen flex flex-col ＋ <main> に flex-1 で、フッターが常に最下部に張り付く
  • ナビ・フッターは bg-secondary text-white
  • ナビ右の「Register｜Log in」は route('register') / route('login') へ

確認：login画面を開き、青ナビ・クリーム背景・青フッターが出ればOK（カード中身はまだBreezeのまま）。

2

カードの中身を組む（login.blade.php）

カード内を上から順に。

要素ポイント
見出し「Log in」太字・中央寄せ（text-2xl font-bold text-center 等）
サブ文「アカウントにログインしてください」text-gray-500 text-center
E-MAILラベル＋必須ラベル＋赤い必須バッジを横並び
メール入力欄薄グレー背景・角丸。placeholder="example@email.com"
PASSWORDラベル＋必須同上
パスワード入力欄同上。下に「英数字8文字以上」text-sm text-gray-500
Log inボタンw-full の青ボタン（bg-secondary text-white）
新規登録リンク「アカウントをお持ちでないですか？新規登録はこちら」/ リンク部は青

消さないこと：フォームの @csrf、name="email" / name="password"、 <x-input-error>（バリデーションエラー表示）は機能に必須。見た目を変えてもこれらは残す。

3

必須バッジをコンポーネント化（任意だが推奨）

「必須」バッジは login と register で繰り返すので、部品化しておくと再利用できる。

<!-- components/required-badge.blade.php -->
<span class="px-2 py-0.5 bg-red-100 text-red-600 text-xs rounded">必須</span>

使うときは <x-required-badge /> と書くだけ。

4

ロゴの差し替え

  • ロゴ画像を public/images/logo.png に配置
  • components/application-logo.blade.php を <img src="{{ asset('images/logo.png') }}" ...> に差し替え
  • 画像が未用意なら、当面テキスト「Readoo!」で進めてOK
5

最終確認

  • 青ナビ・クリーム背景・青フッターが表示される
  • メール・パスワード入力欄が正しく表示され、必須バッジが出る
  • 実際にログインできる（@csrf / name属性が残っている）
  • バリデーションエラーが該当項目に表示される（誤入力で確認）
  • 「新規登録はこちら」でregister画面へ遷移できる
  • register画面にも同じ枠（ナビ・背景・フッター）が出ている

規約リマインド

  • Tailwindクラス順序：レイアウト → サイズ → スペース → 色 → テキスト → その他
  • Bladeにはロジックを書かない（表示のみ）
  • 退会など危険操作のボタンは別途 赤系（danger） を維持（今回は対象外）
  • コミットはConventional Commits（例：feat: ログイン画面をブランドデザインに刷新）

次にやること（このあとの画面）

ログインができたら、同じ枠を活かして register画面 → プロフィール/マイページ → ナビゲーション（ログイン後） の順に展開していく。
