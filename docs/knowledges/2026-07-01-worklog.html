① 知識一覧ビュー（Figma準拠）

独立した知識一覧は本をまたぐため、「どの本の知識か」が一目で分かる列構成にした。

列内容備考
タイトル$knowledge->title太字。リンクにはせずプレーン表示（Figma通り）
参照元$knowledge->book->titlewith('book')済みなのでN+1なし
該当箇所book_page ?? '-'「P.XX」表記
タグtag1〜3 を青枠ピルarray_filter でnull除外
詳細show への導線v1の方針（編集・削除は詳細に集約）と整合
{{-- タグ：入っているものだけピル表示 --}}
@foreach (array_filter([$knowledge->tag1, $knowledge->tag2, $knowledge->tag3]) as $tag)
    <span class="... rounded-full border border-blue-200 bg-blue-50 text-blue-600 text-xs">{{ $tag }}</span>
@endforeach
件数行はページネータのメソッドで作る。 全 {{ $knowledges->total() }} 件中 {{ $knowledges->firstItem() }}〜{{ $knowledges->lastItem() }} 件。 firstItem()/lastItem() は空のとき null を返すが、ここは @else（空でない）側なので安全。 件数行・テーブル・ページネーションを1つの白カードにまとめてFigmaに合わせた。
ページネーション補足。 $knowledges->links() はLaravel標準。画面が広いと英語の「Showing X to Y of Z」も出る。 Figmaの「< Prev 1 2 … Next >」だけに寄せたい場合は vendor:publish --tag=laravel-pagination でビューをカスタムできる（今は標準のまま）。

② KnowledgeFactory：親を連鎖生成する

return [
    'book_id'   => Book::factory(),                       // ← 親の本も自動生成
    'title'     => fake()->sentence(3),
    'book_page' => 'P.'.fake()->numberBetween(1, 500),
    'content'   => fake()->paragraph(),
    'tag1'      => fake()->word(),
    'tag2'      => null,
    'tag3'      => null,
];
  • 'book_id' => Book::factory() … BookFactory の 'user_id' => User::factory() と同じ発想。 User → Book → Knowledge が芋づる式に揃う
  • book_id は Fillable 除外だがFactoryは内部で unguarded なのでセットできる（user_id と同じ）
  • tag2 / tag3 => null … 任意項目のデフォルトは空。3つ揃えたいときは生成時に上書き
  • 命名規約（App\Models\Knowledge ⇄ Database\Factories\KnowledgeFactory）で自動解決。追加設定不要

③ 表示確認：ダミーは「ログイン中ユーザーの本」に紐づける

今日いちばんの勘所。 知識一覧は whereHas('book', user_id 一致) でログイン中ユーザーの知識だけを出す。 だからダミーデータも「ブラウザでログインしているアカウントの本」に紐づけて作らないと、一覧に出ない。 
// tinker
$user = \App\Models\User::where('email', 'juri.mino1023@gmail.com')->first();
$book = \App\Models\Book::factory()->for($user)->create(['title' => '確認用の本']);

\App\Models\Knowledge::factory()->count(5)->for($book)->create();
// タグ3つ揃ったパターンも（ピル確認用）
\App\Models\Knowledge::factory()->count(2)->for($book)->create(['tag2' => 'サンプル', 'tag3' => '重要']);

→ /knowledges を開いて、件数行・参照元・該当箇所・タグのピル・レイアウト崩れなしを確認 OK。

豆知識：退会テストの名残でユーザーは4件存在（id1は論理削除＝メール退避 ..._deleted_1、id4が同じメールで再登録）。 アクティブは yamada@test.com と juri.mino1023@gmail.com の2つ。 退会＝ユーザー論理削除＋メール退避の設計が実データでも確認できた。
