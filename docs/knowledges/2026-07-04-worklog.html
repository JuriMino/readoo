1. 編集ビュー（edit.blade.php）

create を土台に、以下の点を更新用へ差し替えた。

箇所createeditなぜ
HTTPメソッドPOST のみ@method('PATCH') 追加resource の update は PATCH
actionknowledges.storeknowledges.update, $knowledge更新先
初期値old('field')old('field', $knowledge->field)初回は既存値、差し戻し時は old
参照元（本）ドロップダウン読み取り専用の表示update で book_id 除外＝変更不可
from 隠しフィールドありなしupdate は常に show へ戻る（分岐不要）
Cancelfrom で分岐knowledges.show 固定編集元は必ず詳細画面
参照元（本）は編集不可（v1確定）

update() は $request->safe()->except('book_id') で book_id を除外し、KnowledgeRequest も book_id のバリデーションを isMethod('post')（作成時）だけに限定している。編集画面で本を付け替える操作は update で無視されるため、ドロップダウンではなく読み取り専用の表示にした（ユーザーの誤操作防止）。別の本への付け替えは将来スコープ。

ループ内の変数変数（$knowledge->$name）

タグは @foreach (['tag1'=>..., 'tag2'=>..., 'tag3'=>...]) で回している。old($name, $knowledge->$name) の $knowledge->$name は「変数変数」で、$name が 'tag1' なら $knowledge->tag1 を参照する。
