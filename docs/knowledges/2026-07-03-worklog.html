1. 知識詳細ビュー（show.blade.php）

books/show.blade.php の流儀（パンくず・編集/削除アイコン・x-modal 削除確認）に揃え、テーマ色は知識＝青で作成。コントローラは ['knowledge' => $knowledge] のみ渡し、$knowledge->book は参照時に遅延ロードされる（Policy の view でも book を見るのでどのみち読まれる）。

Figma に合わせた構成修正

最初の叩き台は「参照元／知識の内容／タグ」の構成だったが、共有された Figma は books/show と対になる構成だったため作り直した。

セクション中身
基本情報タイトル（全幅）／ 参照元（本）＋該当箇所（2カラム）／ 登録日
本文情報内容（whitespace-pre-wrap で改行保持）
タグタグピル（array_filter で null 除去、空なら「登録されていません」）
参照元（本）はリンクに 
<a href="{{ route('books.show', $knowledge->book) }}" ...>{{ $knowledge->book->title }}</a>

「この知識はどの本から得たか」を1クリックで辿れる。知識詳細 ⇄ 本の詳細を相互に行き来できるようになった。

「紐づく行動」セクションは今日は作らない

Figma には一番下にオレンジの「紐づく行動」セクション（＋ new Action）があるが、これは 行動（Action）機能そのもの。モデルもリレーションも未実装で、$knowledge->actions を今書くとエラーになる。Action 機能の実装時に実データで追加する（二度手間回避）。

削除の扱い（v1）

  • 削除できるのはこの詳細画面のみ（x-modal の確認 → @csrf + @method('delete')）
  • 削除後はコントローラ側で親の本（books.show）へリダイレクト＋ステータス表示
  • 複数選択削除（一覧から）は将来の v2 で対応予定

2. 写経ミスの修正

いずれも Tailwind が無効クラスを黙って無視するため、エラーは出ず見た目だけ崩れるタイプ。

行誤正影響
2max-w-3-xlmax-w-3xl横幅が制限されない
7class="mt-1"（パンくず区切り）class="mx-1"「>」の左右余白が出ない
12fount-boldfont-bold見出しが太字にならない
31py-6（見出し）py-3基本情報の見出しだけ高さが不揃い（他は py-3）
指示の切り分けメモ

py-3 → py-6 は「<dl>（中身の余白）」だけの意図だったが、見出し <h2> にも適用してしまい高さがずれた。セクション見出しは3つとも py-3 で統一。
