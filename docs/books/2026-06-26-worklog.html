本日のサマリー

Figmaモックに合わせて詳細画面（show）を実装し、削除機能（destroy）を x-modal による確認モーダル経由でつないだ。詳細表示・削除ともにブラウザで動作確認OK。 削除は論理削除（SoftDeletes）で、一覧から消えるが deleted_at に退避される挙動を確認。 これで index / create / show ＋削除が揃い、CRUDの「R・D」が完成。残りは edit（編集）＋ update。

  • show（詳細）：基本情報 <dl> ＋ 本文情報、編集/削除アイコン
  • 削除：x-modal 確認モーダル → @method('delete') で destroy へ
  • SoftDeletes による論理削除を確認（物理削除ではない）
  • 削除後 → 一覧リダイレクト → フラッシュ表示

① show（詳細表示）

ルートモデルバインディングで Book $book を受け取り、Policy（view）で 本人の本だけ表示。基本情報は <dl>（定義リスト）で「項目名＋値」を表現した。

押さえどころ

記述意味
<dl> / <dt> / <dd>定義リスト。dt＝項目名（タイトル等）、dd＝値。意味的に「ラベルと値の対」を表す正しいタグ
grid grid-cols-2grid と grid-cols-2 はセットで必須。grid が無いと2カラムにならない（本日も踏んだ → 後述）
col-span-2タイトルだけ全幅。グリッド内で2カラム分ぶち抜く
whitespace-pre-wrap要約・メモの改行をそのまま表示。通常HTMLは改行を無視するので必須
$book->started_at?->format('Y-m-d') ?? '-'ヌル安全演算子で未設定はダッシュ表示（index と同じ作法）
編集/削除アイコンの違い：
  • 編集 = <a href="{{ route('books.edit', $book) }}">（画面遷移するのでリンク）
  • 削除 = <button type="button" x-on:click="$dispatch('open-modal', ...)">（モーダルを開くだけでその場では送信しない）
「遷移するもの＝a」「その場で何か起こすもの＝button」の使い分け。

② 削除機能（モーダル ＋ destroy）

誤操作防止のため確認モーダルを挟む。Breeze 同梱の x-modal（Alpine.js）を、 退会フォーム（profile）と同じパターンで流用した。

{{-- 詳細画面のアイコン側：モーダルを開くだけ --}}
<button type="button" x-data="" x-on:click.prevent="$dispatch('open-modal', 'confirm-book-deletion')"> … </button>

{{-- モーダル本体：実際に送信するフォーム --}}
<x-modal name="confirm-book-deletion" focusable>
    <form action="{{ route('books.destroy', $book) }}" method="post">
        @csrf
        @method('delete')   {{-- ← HTMLはDELETEを送れないのでPOST＋偽装 --}}
        …
    </form>
</x-modal>
記述意味
$dispatch('open-modal', 'confirm-book-deletion')名前付きでモーダルを開くAlpineイベント。name と一致させる
$dispatch('close')Cancelボタンで閉じる
@method('delete')HTMLフォームはGET/POSTしか送れない。_method=DELETE を仕込んでLaravelに「DELETEとして扱わせる」
@csrfCSRF対策トークン。POST系は必須（無いと419）
SoftDeletes（論理削除）： $book->delete() はレコードを消さず deleted_at に日時を入れるだけ。 以降 $user->books() の通常クエリからは自動で除外され、一覧から消える。 ルートモデルバインディングも論理削除済みは404扱いになるため、削除済みURLを直叩きしても詳細は開けない。 物理削除が必要なのは「退会時」だけ（後述の残課題 forceDelete()）。

③ 確認中に見つかった要修正点（show.blade.php）

写経した show.blade.php を見直したところ、画面では気づきにくい不具合が残っていた。 いずれも PHP エラーにはならず、表示や保存値で初めて分かるタイプ。

症状原因修正
メモ欄に「要約」と同じ内容が出る（機能バグ） メモの dd が {{ $book->summary }}（コピペミス） {{ $book->memo ?? '-' }}
基本情報が2カラムにならず縦並び <dl> に grid-cols-2 はあるが grid が無い grid grid-cols-2
パンくずの文字色が効かない rexr-gray-400（タイポ） text-gray-400
削除アイコンのホバー色が効かない hovder:text-red-600（タイポ） hover:text-red-600
本文情報の見出し色が効かない text-grau-700（grau） text-gray-700
昨日の教訓が再発： grid-cols-2 抜けの grid、grau / rexr / hovder のタイポ。 「スタイルが効かない＝まずクラス名のスペル」「grid-cols-* は grid とセット」を改めて徹底する。 Tailwindは存在しないクラスを黙って無視するため、レビュー（人の目 or 差分確認）でしか拾えないのがポイント。
