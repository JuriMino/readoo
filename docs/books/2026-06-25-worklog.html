① 表示ラベルの一元管理（Bookモデルの定数）

ジャンル/ステータスの日本語ラベルは index・create・show の3画面で必要。毎回マップを書くと重複するので、 Bookモデルに定数として集約した。

class Book extends Model
{
    public const STATUSES = [
        'unread' => '未読', 'reading' => '読書中', 'finished' => '読了',
    ];
    public const GENRES = [
        'business' => 'ビジネス', 'self_help' => '自己啓発', /* … 全13種 */ 'other' => 'その他',
    ];
効果（DRY）：
  • 一覧表示：Book::GENRES[$book->genre] で「ビジネス」を取得
  • セレクト選択肢：@foreach (Book::GENRES as $value => $label) で 選択肢を自動生成
  • ジャンル追加時もモデル1か所を直すだけで全画面に反映。値（DB保存値）と表示（日本語）が常に一致する。

③ create（登録フォーム）

基本情報／読書期間／本文情報の3セクション。必須は x-required-badge、任意は新規作成した x-optional-badge を使用。

押さえどころ

  • セレクトを定数から生成：@foreach (Book::GENRES as $value => $label)。value＝DB保存値、表示＝日本語。
  • @selected(...)：条件trueで selected 属性を出力するBlade構文。バリデーションエラーで戻ったとき前回選択を復元。
  • old('status', 'unread')：第2引数は old値が無いときの初期値。初回表示で「未読」を選択状態に。
  • textareaのold値は「タグの間」に置く（input は value=""）。>{{ old('summary') }}</textarea> のように空白・改行を入れず密着させる（間の空白が初期値に混入するため）。
  • Cancelは <a>、Createは <button type="submit">。送信するのはCreateだけ。

④ 今日詰まった不具合と修正（学び多め）

本日は写経時のタイポ・タグ閉じ位置由来の不具合が連発した。いずれも「PHPの構文エラー」にはならず、 画面表示で初めて気づくタイプ。

症状原因修正
登録画面で Undefined variable $messages（500） x-input-error を :message（単数）で渡していた（9か所） :messages（複数）に統一
見出し・フォームが全幅で max-w-3xl が効かない </div> の閉じ位置が早く、中央寄せ枠がパンくずだけを囲んでいた 閉じ </div> を </form> の後（末尾）へ移動
2カラムが横並びにならない gird-cols-2 / gap06 のタイポ grid-cols-2 / gap-6
入力欄の背景グレーが出ない bg-grau-50（grau） bg-gray-50
出版社が空で登録できない 「任意」項目の publisher に required を付けていた required を削除
ステータスの初期選択が未読にならない old('status', 'old') と存在しない値を初期値に old('status', 'unread')
今日の最大の学び：Tailwindは「存在しないクラスを黙って無視する」。 タイプミスしてもエラーは出ず、ただ「効かない」だけ。だから 「スタイルが効かない＝まずクラス名のスペルを疑う」が鉄則。 gird / grau / gap06 はその典型。
もう一つの学び：max-w-* や mx-auto が効かないように見えるとき、 クラスではなく「対象がその枠の“外”に出ている」ことが多い。 原因は </div> の閉じ位置ズレ。ネストが深いBladeは閉じタグに {{-- セクション終わり --}} 等のコメントを添えると迷子になりにくい。
再発した罠：x-input-error は :messages（複数形）。 input-error.blade.php は @props(['messages']) でデフォルトが無いため、 渡し忘れ／単数形だと即500。保険として @props(['messages' => null]) を入れておくとクラッシュしない。 （プロフィール画面でも踏んだのと同じパターン）
