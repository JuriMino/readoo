② ステータスのバッジ化 ＋ 登録日

モックに合わせ、ステータスを錠剤型バッジに。さらに登録日（created_at）を追加し、 日付3つを横3列に並べるため基本情報の <dl> を grid-cols-6 化した。

<dd class="mt-1">
    <span class="inline-block px-3 py-1 rounded-full text-xs font-bold {{ $statusStyles[$book->status] }}">
        {{ \App\Models\Book::STATUSES[$book->status] }}
    </span>
</dd>
記述意味
<span> + inline-blockdd はブロック要素で全幅。span＋inline-block で中身の幅だけに収め、帯にならないバッジへ
rounded-full錠剤型の丸み（rounded-lg は角丸四角）。一覧 index のバッジと統一
grid grid-cols-62列（col-span-3）も3列（col-span-2）も割れる土台。タイトルは col-span-6
$book->created_at->format('Y-m-d')created_at は必ず値があるので ?-> 不要。任意項目の started_at 等は ?-> が要る、の対比で理解
col-span の配分（grid-cols-6）： タイトル＝col-span-6（全幅）／ 著者・出版社・ジャンル・ステータス＝col-span-3（2列）／ 読書開始日・読了日・登録日＝col-span-2（3列）。6を分母にすると2列・3列を1つのグリッドで両立できる。

③ edit（編集フォーム）＋ update

create とほぼ同形。違いは3点だけ。

箇所createedit
送信先route('books.store')route('books.update', $book)
メソッドPOSTPOST ＋ @method('PATCH')
初期値old('title')old('title', $book->title)
セレクト@selected(old('status','unread') === $value)@selected(old('status', $book->status) === $value)
日付old('started_at')old('started_at', $book->started_at?->format('Y-m-d'))
最重要3点：
  • old('title', $book->title) … 第2引数にDB値。初回＝DB値、エラーで戻ったとき＝入力中の値、を両立。全項目この形。
  • @method('PATCH') … HTMLはPATCHを送れないのでPOST＋偽装（削除の @method('delete') と同じ理屈）。
  • $book->started_at?->format('Y-m-d') … <input type="date"> の value は YYYY-MM-DD 形式必須。Carbonをformatで整形（nullなら空）。
将来のリファクタ候補：create と edit はフォーム部分がほぼ同一。実務では books/partials/_form.blade.php に切り出して両方から @include するのが定石。 今は「まず動かす」優先で別ファイルにしている。CRUD一段落後の練習として共通化するとよい。
