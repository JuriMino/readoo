Laravel Breeze 導入手順

Readoo! — 認証機能（セッション認証）のセットアップガイド

前提環境

項目内容
Laravel13.14.0
PHP8.5.7（要件 8.3 以上）
フロント構成Blade + Tailwind CSS（モノリシック）
認証Laravel Breeze（セッション認証）
Docker サービスweb(Nginx) / app(PHP-FPM) / db(MySQL 9.4)
重要： このプロジェクトは Docker 構成のため、すべての php / composer / npm コマンドは app コンテナ内で実行します。 コマンドの先頭に docker compose exec app を付けるのが基本です。

導入ステップ

1作業ブランチを作成

main への直接 push は禁止です。Issue を立てた上で作業ブランチを切ります。

git switch -c feature/2-auth-breeze

ブランチ名は feature/(issue番号)-(概要) の規約に合わせてください（issue番号は実際のものに置き換え）。

2コンテナを起動

docker compose up -d
docker compose ps   # web / app / db が Up になっているか確認

3Breeze を Composer で導入

Breeze 本体（開発時のみ使うスキャフォールドツール）を --dev でインストールします。

docker compose exec app composer require laravel/breeze --dev

4Breeze をインストール（Blade スタックを選択）

認証用の Controller・Blade・ルートを生成します。スタックは必ず blade を指定します。

docker compose exec app php artisan breeze:install blade
対話プロンプトについて： テスト用フレームワーク（Pest / PHPUnit）の選択を聞かれます。好みで選んで構いません。 引数で blade を渡しているため、スタック選択（Vue/React/API）はスキップされます。

生成される主なもの：

種別場所
認証コントローラapp/Http/Controllers/Auth/
プロフィールapp/Http/Controllers/ProfileController.php
ビューresources/views/auth/、profile/、dashboard.blade.php
ルートroutes/auth.php（web.php から読み込み）

5フロント資産をビルド

Breeze は Tailwind / Vite の設定も追加します。Node 依存をインストールしてビルドします。

docker compose exec app npm install
docker compose exec app npm run build   # 本番ビルド
# 開発中はホットリロードを使う場合：
# docker compose exec app npm run dev

6マイグレーションを実行

users など認証に必要なテーブルを作成します。

docker compose exec app php artisan migrate
確認： .env の DB 接続先が Docker の db サービスを指しているか （DB_HOST=db / DB_DATABASE=readoo など）事前にチェックしてください。

7動作確認

ブラウザで以下にアクセスし、登録・ログイン・ログアウトが通るか確認します。

http://localhost:8080/register
http://localhost:8080/login

未ログインで保護ページにアクセスした際、ログイン画面へリダイレクトされれば正常です。

導入後に必要なカスタマイズ Readoo! 固有

Breeze は「足場」を置くだけなので、本プロジェクトの仕様に合わせて次の調整が必要です。

① 退会の論理削除対応（必須）

仕様では users は論理削除（SoftDeletes）。Breeze デフォルトの退会は物理削除のため、 User モデルへの SoftDeletes 適用・deleted_at カラム追加・退会処理の改修が必要です。

② パスワードリセットの扱い（要判断）

v1 スコープ外（メール送信を伴うリセット）ですが、Breeze は関連ルート・画面を生成します。 「残す／ルートを外す／ビューを削除」のいずれにするか決めてください。

③ ブランドカラーの適用

生成ビューは素の Tailwind。メイン #E8510A（オレンジ）／サブ #2D5BE3（ブルー）を tailwind.config.js やコンポーネントへ反映します。

④ アクセス制御

保護したいルートを auth ミドルウェアでくくります。未ログイン→ログイン画面リダイレクトは デフォルト挙動のため改修不要。他ユーザーのデータへのアクセス防止（認可）は別途実装が必要です。

完了チェックリスト

  • 作業ブランチを作成した
  • Breeze（blade）をインストールした
  • npm でフロント資産をビルドした
  • マイグレーションが成功した
  • 登録・ログイン・ログアウトが動作した
  • 退会の SoftDeletes 対応方針を決めた
  • パスワードリセットの扱いを決めた
  • ブランドカラー適用方針を決めた
